  <div style={{marginBottom: "60px"}}>
    <h2 style={{fontSize: "28px", fontWeight: "700", fontFamily: "Roboto, Arial, sans-serif", color: "#202124", marginBottom: "12px"}}>
      Define once, use everywhere
    </h2>
    <p style={{color: "#5f6368", fontSize: "16px", marginBottom: "24px", fontFamily: "Roboto, Arial, sans-serif"}}>
      Write a <code>.proto</code> schema file, then generate type-safe code in any supported language.
    </p>

```protobuf addressbook.proto
syntax = "proto3";
package tutorial;

import "google/protobuf/timestamp.proto";

message Person {
  string name  = 1;
  int32  id    = 2;
  string email = 3;

  enum PhoneType {
    MOBILE = 0;
    HOME   = 1;
    WORK   = 2;
  }

  message PhoneNumber {
    string    number = 1;
    PhoneType type   = 2;
  }

  repeated PhoneNumber phones       = 4;
  google.protobuf.Timestamp last_updated = 5;
}

message AddressBook {
  repeated Person people = 1;
}
```

  {/* Feature Cards */}
  <h2 style={{fontSize: "28px", fontWeight: "700", fontFamily: "Roboto, Arial, sans-serif", color: "#202124", marginBottom: "32px"}}>
    Why Protocol Buffers?
  </h2>

  <CardGroup cols={2}>
    <Card title="Language Neutral" icon="globe">
      Write your schema once in `.proto` format and generate idiomatic code for C++, Java, Python, Go, C#, Ruby, PHP, Rust, Objective-C, and more.
    </Card>
    <Card title="Compact and Fast" icon="bolt">
      Binary encoding is 3–10x smaller than XML and JSON, with significantly faster serialization and deserialization throughput.
    </Card>
    <Card title="Schema Evolution" icon="arrows-rotate">
      Add new fields without breaking existing code. Old clients ignore unknown fields; new clients handle missing old fields gracefully.
    </Card>
    <Card title="Strong Typing" icon="shield-check">
      Every message field has an explicit type. Generated code gives you compile-time type safety in statically-typed languages.
    </Card>
    <Card title="gRPC Integration" icon="network-wired">
      First-class support for defining RPC service interfaces. Used by Google and thousands of companies for inter-service communication.
    </Card>
    <Card title="Extensible" icon="puzzle-piece">
      Write your own `protoc` plugins to generate custom code. Custom options let you annotate schemas with application-specific metadata.
    </Card>
  </CardGroup>
