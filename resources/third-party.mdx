<Warning>
  These projects are not affiliated with or endorsed by Google unless explicitly marked. They are community-maintained and quality varies. Use them at your own risk, and verify production-readiness before adopting.
</Warning>

If you maintain a project that should be listed here, [send a pull request](https://github.com/protocolbuffers/protobuf/pulls) to update `docs/third_party.md` in the repository.

## Language runtimes

Google ships official runtimes for C++, Java, Python, Go, C#, Ruby, PHP, Objective-C, and Rust. The projects below provide support for additional languages or alternative implementations.

<AccordionGroup>
  <Accordion title="C and C++">
    - [protobuf-c](https://github.com/protobuf-c/protobuf-c) — Pure C runtime
    - [nanopb](https://koti.kapsi.fi/jpa/nanopb/) — Small code-size C implementation for embedded systems
    - [pbc](https://github.com/cloudwu/pbc/) — C library focused on performance
    - [upb](https://github.com/haberman/upb/wiki) — Micro protobuf library in C
    - [protobluff](https://github.com/squidfunk/protobluff) — C implementation
    - [pbtools](https://github.com/eerimoq/pbtools) — C code generator
    - [simple-protobuf](https://github.com/tonda-kriz/simple-protobuf) — Alternative C++ implementation
    - [EmbeddedProto](https://EmbeddedProto.com) — C++ for embedded targets

## RPC implementations

[gRPC](http://www.grpc.io/) is Google's official RPC framework for Protocol Buffers. The following projects provide alternative or complementary RPC support.

<CardGroup cols={2}>
  <Card title="gRPC" icon="server" href="https://github.com/grpc/grpc">
    Google-official implementation for C++, Node.js, Python, Ruby, Objective-C, PHP, and C#.
  </Card>
  <Card title="Twirp" icon="plug" href="https://github.com/twitchtv/twirp">
    Simple RPC framework supporting multiple languages.
  </Card>
  <Card title="ZeroC Ice" icon="snowflake" href="https://zeroc.com/products/ice">
    Multi-language RPC middleware.
  </Card>
  <Card title="protojure" icon="code" href="https://protojure.github.io">
    gRPC and protobuf for Clojure.
  </Card>
  <Card title="elixir-grpc" icon="bolt" href="https://github.com/elixir-grpc/grpc">
    gRPC implementation for Elixir.
  </Card>
  <Card title="gRPC-haskell" icon="lambda" href="https://github.com/awakesecurity/gRPC-haskell">
    Haskell bindings for gRPC.
  </Card>
  <Card title="grpc-federation" icon="diagram-project" href="https://github.com/mercari/grpc-federation">
    Generate a gRPC server from custom proto options.
  </Card>
  <Card title="icerpc-csharp" icon="ice-cream" href="https://github.com/icerpc/icerpc-csharp">
    Modern RPC framework for C#.
  </Card>
</CardGroup>

## Utilities

<AccordionGroup>
  <Accordion title="Linting and validation">
    - [protoc-gen-lint](https://github.com/ckaznocha/protoc-gen-lint) — Linter for `.proto` files
    - [protolint](https://github.com/yoheimuta/protolint) — Enforce style and conventions
      - [vscode-protolint](https://github.com/plexsystems/vscode-protolint)
      - [intellij-protolint](https://github.com/yoheimuta/intellij-protolint)
      - [vim-protolint](https://github.com/yoheimuta/vim-protolint)
    - [super-linter](https://github.com/github/super-linter) — Protocol Buffer lint as a GitHub Action
    - [Protolock](https://github.com/nilslice/protolock) — Prevent backward-incompatible changes to `.proto` files
  </Accordion>

  <Accordion title="Code generation helpers">
    - [protoc-gen-fieldmask](https://github.com/idodod/protoc-gen-fieldmask) — Generate static-type fieldmask paths
    - [protoc-gen-bq-schema](https://github.com/GoogleCloudPlatform/protoc-gen-bq-schema) — Generate BigQuery schema files
    - [Proto Boiler](https://github.com/in4lio/protoboiler) — Generate boilerplate from `.proto` files using templates
    - [node-protoc-plugin](https://github.com/konsumer/node-protoc-plugin) — Write protoc plugins in Node.js
    - [go-protoparser](https://github.com/yoheimuta/go-protoparser) — Parse `.proto` files in Go
    - [rubberbuf](https://github.com/s-expresso/rubberbuf) — Transpile protobuf into AST (Clojure/ClojureScript)
  </Accordion>

  <Accordion title="JSON and alternate encodings">
    - [protobuf-java-format](https://code.google.com/p/protobuf-java-format/) — JSON, XML, HTML encodings for Java protobufs
    - [protobuf-codec](https://github.com/sijuv/protobuf-codec) — Another JSON encoder/decoder for Java
    - [pbjson](https://github.com/yinqiwen/pbjson) — C++ JSON serialization/deserialization
    - [py-pb-converters](https://github.com/NextTuesday/py-pb-converters) — Python scripts to convert between protobuf and JSON
    - [protobuf-decoder-explainer](https://github.com/jamesdbrock/protobuf-decoder-explainer) — Web tool to decode and explain binary protobuf payloads
  </Accordion>

  <Accordion title="Testing and schema management">
    - [hypothesis-protobuf](https://github.com/CurataEng/hypothesis-protobuf) — Property-based testing utility for Python/Hypothesis
    - [protobuf_spec](https://github.com/connamara/protobuf_spec) — RSpec matchers and Cucumber step definitions
    - [protobuf-dynamic](https://github.com/os72/protobuf-dynamic) — Dynamic schema creation (Java)
    - [protoc-jar](https://github.com/os72/protoc-jar) — Multi-platform executable JAR and Java API for `protoc`
    - [protobuf-el](https://github.com/protobufel/protobuf-el) — Java EE7 Expression Language parser and builder
    - [ProfaneDB](https://profanedb.gitlab.io) — A Protocol Buffers database
  </Accordion>

  <Accordion title="Network tools">
    - [Wireshark/Ethereal plugin](https://code.google.com/p/protobuf-wireshark/) — Packet sniffer plugin for decoding protobuf messages
    - [protobuf-nginx](https://github.com/dbcode/protobuf-nginx/) — Protobuf support for nginx
  </Accordion>
</AccordionGroup>
